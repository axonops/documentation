    =======================

    Restore a single node (total node loss but same IP address)

    1. Ensure Cassandra is stopped on the new node and that its data directories are all empty systemctl stop cassandra rm -rf /var/lib/cassandra/commitlog/ /var/lib/cassandra/data/ /var/lib/cassandra/hints/ /var/lib/cassandra/saved_caches/
    2. Allow the AxonOps user to the Cassandra data directory chmod -R g+w /var/lib/cassandra/data
    3. Start axon-agent if it is not already running systemctl start axon-agent
    4. Trigger a REMOTE restore on only this node from the AxonOps dashboard
    5. Once the restore has completed, fix the permissions on the Cassandra data directories chown -R cassandra.cassandra /var/lib/cassandra/data
    6. Remove write permission for the AxonOps user chmod -R g-w /var/lib/cassandra/data
    7. Start Cassandra on the restored node
    8. Run a repair (or let AxonOps deal with it)

    =======================

    Restore a single node with a different IP (total node loss, IP address changed)

    1. Get the agent ID for the old node from AxonOps Dashboard or from /var/lib/axonops/hostId on the old node
    2. Ensure Cassandra is stopped on the new node and that the data directories are all empty systemctl stop cassandra rm -rf /var/lib/cassandra/commitlog/ /var/lib/cassandra/data/ /var/lib/cassandra/hints/ /var/lib/cassandra/saved_caches/
    3. Allow the AxonOps user to the Cassandra data directories chmod -R g+w /var/lib/cassandra/data
    4. Stop axon-agent if it is running systemctl stop axon-agent
    5. Remove all files and directories in /var/lib/axonops rm -rf /var/lib/axonops/*
    6. Start axon-agent with the old node's host ID echo 'old-host-id' >/var/lib/axonops/hostId systemctl start axon-agent
    7. Trigger a REMOTE restore on only this node from the AxonOps dashboard
    8. Once the restore has completed, fix the permissions on the Cassandra data directories chown -R cassandra.cassandra /var/lib/cassandra/data chmod -R g-w /var/lib/cassandra/data
    9. Remove system keyspace files (these will be recreated during bootstrap): system, system_auth, system_distributed, system_schema, system_traces
    10. Configure Cassandra to replace the old node by adding this line to cassandra-env.sh (replace 1.2.3.4 with the IP address of the old node): JVM_OPTS="$JVM_OPTS -Dcassandra.replace_address=1.2.3.4 -Dcassandra.allow_unsafe_replace=true"
    11. Disable auto-bootstrap on the replacement node by adding this line to cassandra.yaml: auto_bootstrap: false
    12. Start Cassandra on the restored node
    13. Remove or comment-out the lines added to cassandra.yaml and cassandra-env.sh so they are ready for the next restart

    =======================

    Whole cluster restore (Rolling back the entire cluster to a backup)

    1. Stop cassandra on all nodes, leave the AxonOps agent running
    2. Clear the Cassandra data from all nodes rm -rf /var/lib/cassandra/commitlog/ /var/lib/cassandra/data/ /var/lib/cassandra/hints/ /var/lib/cassandra/saved_caches/
    3. Allow the axonops user to write to the Cassandra data directories chmod -R g+w /var/lib/cassandra/data
    4. Trigger a REMOTE restore from AxonOps dashboard
    5. Once the restore has completed, fix the permissions on the Cassandra data directories chown -R cassandra.cassandra /var/lib/cassandra/data
    6. Remove write permission for the AxonOps user chmod -R g-w /var/lib/cassandra/data
    7. Start Cassandra on one node at a time, seeds first

    =======================

    Whole cluster restore (All nodes lost but cluster recreated with same IPs)

    For each node in the new cluster: 1. Ensure Cassandra is stopped and that its data directories are all empty systemctl stop cassandra rm -rf /var/lib/cassandra/commitlog/ /var/lib/cassandra/data/ /var/lib/cassandra/hints/ /var/lib/cassandra/saved_caches/ 2. Start axon-agent on all nodes if it is not already running systemctl start axon-agent 3. Start a REMOTE restore on all nodes from the AxonOps dashboard 4. Once the restore has completed, fix the permissions on the Cassandra data directories chown -R cassandra.cassandra /var/lib/cassandra/data 5. Remove write permission for the AxonOps user chmod -R g-w /var/lib/cassandra/data 6. Start Cassandra on one node at a time, seeds first
